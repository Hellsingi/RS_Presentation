When a web page is loaded, the browser creates a Document Object Model (DOM).

WHAT IS a DOM?

The Document Object Model (DOM) is a programming interface for HTML and XML documents. It represents the page so that programs can change the document structure, style, and content. The DOM model represents a document with a logical tree. Each branch of the tree ends in a node, and each node contains objects.

DOM representation


<!DOCTYPE html>
<html>
<head>
	<title>My Title</title>
</head>
<body>
	<h1>My Header</h1>
	<a href="http://braika.com">
My Link</a>
</body>
</html>

The principal standardization of DOM was handled by the World Wide Web Consortium, which last developed a recommendation in 2004. WHATWG took over development of the standard, publishing it as a living document. The W3C now publishes stable snapshots of the WHATWG standard.

WHY DO WE NEED DOM?

  • provides a structured representation of the document (a tree)
  • DOM is needed in order to manipulate the page: read information from HTML, create and modify elements.

NODE

All items in the DOM are defined as nodes. Node is an interface from which a number of DOM API object types inherit.

There are 12 different node types, which may have children of various node types:

  1. Element
  2. Attribute
  3. Text
  4. CDATA Section
  5. Entity Reference
  6. Entity
  7. Processing Instruction
  8. Comment
  9. Document
  10. Document Type
  11. Document Fragment
  12. Notation

In practice usually work with 4 of them:

  1. Document – the “entry point” into DOM.
  2. Element nodes – HTML-tags, the tree building blocks.
  3. Text nodes – contain text.
  4. Comment – sometimes we can put the information there, it won’t be shown, but JS can read it from the DOM.

All nodes has methods and properties

  • A property is a value that you can get or set.
  • A method is an action you can do.

DOCUMENT

The Document interface represents any web page loaded in the browser and serves as an entry point into the web page's content, which is the DOM tree. The DOM tree provides functionality globally to the document. The Document interface describes the common properties and methods for any kind of document.

Document

Properties:

  • Document.body
  • Document.contentType
  • Document.documentElement
  • Document.images
  • Document.doctype
  • other Properties

Some examples of properties

let objRef = document.body; 
document.body = objRef;

The Document.body property represents the <body> or <frameset> node of the current document, or null if no such element exists. 

// in HTML: <body id="oldBodyElement"></body>
alert(document.body.id); // "oldBodyElement"	
let aNewBodyElement = document.createElement("body");
aNewBodyElement.id = "newBodyElement";
document.body = aNewBodyElement;
alert(document.body.id); // "newBodyElement"
let fontFaceSet = document.fonts;

The fonts property of the Document interface returns the FontFaceSet interface of the document. 

document.fonts.ready.then(function() {
	// Any operation that needs to be done only after all the fonts
	// have finished loading can go here.
});
let objRef = document.head;

The head read-only property of the Document interface returns the element of the current document. 

<!doctype html>
<head id="my-document-head">
<title>Example: using document.head </title>
</head>
<script>
var theHead = document.head;
console.log(theHead.id); // "my-document-head";
console.log(theHead === document.querySelector("head")); // true
</script>
